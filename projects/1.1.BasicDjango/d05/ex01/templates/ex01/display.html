정적 페이지 처리 과정

Django에서 정적 웹페이지가 사용자에게 어떻게 전달되는지에 대한 과정은 다음과 같습니다:

  1. 요청 수신: 사용자가 웹 브라우저를 통해 특정 URL로 요청을 보냅니다. 예를 들어, 사용자가 ‘/ex01/display’ 페이지를 요청한다고 가정해 봅시다.
  2. URL 라우터: Django의 URL 라우터는 요청받은 URL을 분석하고, 이를 urlpatterns에서 정의된 패턴과 비교하여 해당 요청을 처리할 뷰 함수를 결정합니다.
  3. 뷰 함수: URL 라우터에 의해 선택된 뷰 함수가 호출됩니다. 이 함수는 요청을 처리하고, 응답을 생성하기 위해 필요한 모든 데이터를 처리합니다.
  4. 템플릿 렌더링: 뷰는 응답을 구성하기 위해 템플릿을 사용합니다. 뷰는 템플릿 엔진에게 HTML 파일 내에서 사용할 데이터를 전달합니다.
  5. HTML 파일: 템플릿 엔진은 전달받은 데이터와 HTML 템플릿을 결합하여 최종적인 HTML 응답을 생성합니다.
  6. 응답 전송: 생성된 HTML 응답이 사용자의 웹 브라우저로 전송되어 페이지가 표시됩니다.

이 과정을 통해 Django는 정적 페이지 요청을 효율적으로 처리하며, 개발자는 URL 라우팅, 뷰 함수, 그리고 템플릿을 통해 웹 페이지를 사용자에게 제공할 수 있습니다.
